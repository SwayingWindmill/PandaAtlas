# Panda Atlas Monorepo Draft

## Scope

- Frontend: Next.js (App Router), Tailwind CSS v4, shadcn/ui, map rendering.
- Backend: FastAPI, SQLAlchemy, Alembic, Pydantic.
- Data platform: Supabase Postgres + PostGIS + Auth + Storage.
- Deployment: `web` on Vercel, `api` on a container host (Render/Railway/Fly.io), database on Supabase.

## Runtime Boundaries

- `apps/web` only talks to `services/api` over HTTP.
- `services/api` is the only write path to Postgres and Storage metadata.
- Supabase Auth JWT is verified in FastAPI for admin routes.
- Supabase Storage signed URLs are generated by backend for controlled media access.

## Suggested Build and CI Pipeline

1. Lint/typecheck:
   - `npm run lint:web && npm run typecheck:web`
   - `uv run ruff check . && uv run pytest -q`
2. Schema checks:
   - Run Alembic migration check in CI before deploy.
3. Contract checks:
   - Keep `services/api/openapi/panda-atlas-v1.yaml` synced from FastAPI docs.
4. Deploy:
   - Merge to `main` triggers Vercel for frontend and container deploy for backend.
