## 1. 决策摘要

PandaAtlas 采用 **“个体档案为中心，探索工具作为顶层入口”** 的公共信息架构。

核心闭环是：

> 找到一只熊猫 → 确认关键事实 → 沿家族、足迹、机构或来源继续探索 → 进入另一个真实实体 → 保留上下文继续浏览。

所有公开内容使用语言前缀化规范 URL。个体详情只保留一个可信档案契约，旧生成式详情退出公共信息架构。首页、图鉴、个体档案、家族探索器、地图和机构页可以有不同页面结构，但必须共享导航、语言、可信语义、URL 状态和等价任务规则。

图形界面只增强关系与空间理解，不拥有独占信息或独占操作。地图与谱系的结构化表示始终是一等界面，而不是加载失败时才出现的备用内容。

## 2. 规范性语言

- **必须 / 不得：** 属于路由、导航或核心旅程的硬约束。
- **应该 / 不应该：** 默认执行；偏离时必须记录用户收益、兼容风险与验证方法。
- **可以：** 在不破坏核心任务、可信度、无障碍和 URL 连续性的情况下使用。

## 3. 信息架构原则

### 3.1 个体档案是知识网络的主要阅读节点

PandaAtlas 不是按数据库表机械组织的实体目录。用户可以从搜索、首页、地图、谱系、机构或分享链接进入，但主要阅读节点是熊猫个体档案。

个体档案负责：

- 确认身份；
- 展示关键事实；
- 解释事实状态与来源；
- 提供家族、足迹、机构、媒体与修订入口；
- 把用户带到下一个真实实体。

### 3.2 探索器负责跨对象分析

- 家族探索器负责跨代、跨个体关系浏览与亲缘路径比较。
- 地图探索器负责跨地点、跨时间、跨机构的空间浏览。
- 图鉴负责搜索、筛选、比较和发现。

探索器不得成为查看基础事实的唯一入口。个体档案本身必须提供足以理解当前个体的家族摘要和足迹列表。

### 3.3 首页负责进入，不负责承载全部产品

首页只承担：

1. 解释 PandaAtlas 是什么；
2. 提供快速搜索；
3. 提供真实档案和可信更新入口。

首页不得直接承载完整地图、完整谱系、大量指标、无来源社会证明或长篇产品说明。

### 3.4 公开状态必须可恢复

任何值得分享、刷新、返回或切换语言后继续的状态，都必须有稳定 URL 表达。浏览器历史是产品交互模型的一部分，不是实现细节。

### 3.5 同一任务、同一语义、不同呈现

移动端、桌面端、键盘、读屏、非视觉、减少动画与无 JavaScript 环境共享同一信息架构。不得维护相互矛盾的简化版任务或第二套内容结论。

## 4. 顶层导航

### 4.1 桌面端主导航

主导航按以下顺序呈现：

1. **首页**
2. **熊猫图鉴**
3. **家族**
4. **地图**
5. **我的熊猫**

全局搜索与语言切换是独立的一级能力，不埋入页面内容区。

### 4.2 各入口职责

#### 首页

解释产品、承接首次访问、提供搜索、代表性档案、主要探索方向和最近已发布修订。

#### 熊猫图鉴

承接完整搜索、筛选、排序、分页、比较和无结果建议。默认以熊猫个体为主，同时可以展示机构与场所结果。

#### 家族

进入关系优先的家族探索器。所有真实熊猫节点最终都能进入其规范档案。

#### 地图

进入统一地图探索器，并在其中明确切换“全球机构”“个体足迹”“野生保护”三个场景。

#### 我的熊猫

承接匿名本地收藏和最近浏览，不成为账户中心、社交中心或通知中心。

### 4.3 机构不是首个 Beta 的顶层导航项

机构是重要实体，但用户主要通过以下路径进入：

- 全局搜索；
- 熊猫当前场所；
- 居住或迁移记录；
- 全球机构地图；
- 相关实体链接。

首个 Beta 不把每个数据库实体都提升为顶层导航项，以免削弱“找到并理解一只熊猫”的主线。机构仍拥有规范详情 URL 和完整可分享页面。

### 4.4 移动端导航

移动端通过明确菜单呈现同一组入口，不得删除“家族”“地图”或“我的熊猫”。

要求：

- 搜索保持一级可见或一级可达，不藏在多层菜单中；
- 语言切换清晰可操作；
- 当前页面与当前语言有可感知状态；
- 菜单完全支持键盘、读屏与触控；
- 关闭菜单后焦点返回触发控件。

## 5. 规范路由树

所有公开页面使用 `/{locale}` 前缀。首个 Beta 支持 `zh` 与 `en`。

### 5.1 规范路由

| 语义 | 规范路由 | 说明 |
| --- | --- | --- |
| 首页 | `/{locale}` | 语言化首页 |
| 图鉴与搜索 | `/{locale}/atlas` | 全局搜索提交后的完整结果工作区 |
| 熊猫档案 | `/{locale}/atlas/{pandaSlug}` | 唯一规范个体详情契约 |
| 家族探索器 | `/{locale}/lineage` | 关系摘要、图形谱系与关系路径 |
| 地图探索器 | `/{locale}/map` | 全球机构、个体足迹与野生保护 |
| 机构详情 | `/{locale}/institutions/{institutionSlug}` | 组织实体详情 |
| 场所详情 | `/{locale}/places/{placeSlug}` | 可分享的实际地点摘要或详情 |
| 我的熊猫 | `/{locale}/my-pandas` | 本地收藏与最近浏览 |

### 5.2 个体详情契约收敛

- `/{locale}/atlas/{pandaSlug}` 是唯一公开个体详情。
- `/atlas/{slug}` 的旧生成式详情不得继续作为第二个可索引页面。
- 旧详情中的生成式故事、性格、媒体、位置和推荐不得迁入规范路由，除非经过可信内容规则和发布流程验证。
- 现有测试若依赖旧详情，应随规范契约迁移，而不是反过来保留错误信息架构。

### 5.3 无语言前缀路由

以下无语言前缀地址只承担语言解析和重定向，不承载独立内容：

- `/`
- `/atlas`
- `/atlas/{slug}`
- `/lineage`
- `/map`
- `/global-distribution`
- `/institutions/{slug}`
- `/places/{slug}`
- `/my-pandas`

语言解析顺序：

1. 已保存的明确语言偏好；
2. 可支持的请求语言；
3. 项目默认语言 `zh`。

重定向必须保留可兼容的查询参数和锚点。

### 5.4 旧路由与旧 slug

- 历史 slug 永久重定向到当前规范 slug。
- 一个历史 slug 只能指向原稳定身份，不得重新分配给另一只熊猫。
- `/global-distribution` 永久重定向到对应语言的 `/map`。
- 旧 `/map` 无语言地址只做语言重定向。
- 重定向不得丢失模式、选中实体、时间范围、查询词或章节锚点等兼容状态。

### 5.5 规范标记

每个可索引页面必须具备：

- 自引用 canonical；
- 中英文 `hreflang`；
- 合理的 `x-default`；
- 稳定实体 ID 与 slug 的内部对应；
- 对已下线、合并或改名实体的明确重定向策略。

## 6. 全局搜索与图鉴

### 6.1 一个入口，两个结果层级

全局搜索是所有公开页面都可用的快速入口。

#### 第一层：轻量建议

输入过程中可提供按实体类型分组的建议：

- 熊猫；
- 机构；
- 场所。

建议用于快速到达，不承担完整筛选和结果管理。

#### 第二层：图鉴结果工作区

提交搜索后进入当前语言的 `/{locale}/atlas`。图鉴承担：

- 完整结果；
- 类型切换；
- 筛选；
- 排序；
- 分页；
- 无结果和近似建议；
- 返回或分享同一状态。

### 6.2 搜索覆盖范围

首个 Beta 支持：

- 中文名；
- 英文名；
- 拼音；
- 别名；
- 旧名或历史拼写；
- 外部编号；
- 机构名称；
- 场所名称。

首个 Beta 不把事件、AI 问答、自然语言关系查询或未经审核的编辑专题作为搜索主结果。

### 6.3 结果组织

- 默认结果以熊猫个体为主。
- 混合结果必须明确标注实体类型。
- “机构”和“场所”不得合并成含糊的“地点”。
- 名称匹配、别名匹配和旧名匹配可以解释匹配原因。
- 点击建议可直接进入实体详情。
- “查看全部结果”必须进入保留当前查询的图鉴状态。

### 6.4 无结果

无精确结果时应：

1. 明确说明没有精确命中；
2. 显示可解释的名称或别名近似；
3. 显示相关熊猫、机构或场所；
4. 提供清除部分筛选或返回全部结果的操作。

不得通过伪造“热门推荐”或无来源标签填充空状态。

### 6.5 图鉴 URL 状态

图鉴至少必须表达：

- 查询词；
- 实体类型；
- 关键筛选；
- 排序；
- 页码。

示例：

```text
/{locale}/atlas?q=bao&type=panda&sort=name&page=2
```

具体筛选键和值由后续系统与实施规格确定，但不得使用翻译后的显示文案作为 URL 枚举值。

## 7. 首页信息架构

首页按以下顺序组织。

### 7.1 产品定位与搜索

首屏包含：

- 一句话说明 PandaAtlas 是大熊猫个体、家族与迁移档案；
- 全局搜索；
- 一个清晰的主要动作。

不得使用：

- 无来源用户量；
- 无来源物种或机构统计；
- 空泛保护成效；
- 内部设计过程说明；
- 无实际目标的 CTA。

### 7.2 代表性档案

展示首个 Beta 中经过审核的代表性熊猫。

每个入口只使用：

- 真实名称；
- 合规媒体或明确无图状态；
- 公开档案等级；
- 有来源的关键摘要。

不得使用本地热门分、虚构性格、无来源标签或伪推荐理由。

### 7.3 从关系和地点开始探索

首页提供少量真实任务入口，例如：

- 进入美香—添添家族；
- 浏览全球机构；
- 查看一只熊猫的真实足迹。

每个入口必须对应可完成的产品任务，而不是装饰性专题卡片。

### 7.4 最近档案修订

展示已经发布的：

- 档案变化；
- 核实时间更新；
- 新增来源；
- 关系或居住记录修订。

每条更新显示对象、发布日期和修订摘要。不得把未经审核新闻流作为产品动态。

### 7.5 可信档案说明

简要解释：

- 来源；
- 最后核实日期；
- 不确定性与冲突；
- 媒体许可；
- 修订和版本。

首页只做概览，并链接到完整方法、来源或数据说明。

## 8. 个体档案信息架构

个体档案采用 **“先确认身份，再展开证据与关系”** 的渐进式双层结构。

### 8.1 身份首屏

首屏固定优先呈现：

- 中文名；
- 英文名；
- 拼音；
- 适用的别名或旧称；
- 主图或明确无图状态；
- 一句话人工审核简介；
- 出生信息；
- 性别；
- 当前状态；
- 当前场所及最后核实日期；
- 父亲和母亲；
- 档案等级；
- 收藏、分享和纠错。

长故事、画廊、统计数字或装饰性大图不得挤占关键事实优先级。

### 8.2 稳定章节

正文使用以下稳定章节语义：

1. **概览**
2. **故事**
3. **时间线**
4. **家族**
5. **足迹**
6. **影像**
7. **资料依据与修订**

没有真实、已发布内容的章节不显示空壳导航项。

### 8.3 章节职责

#### 概览

身份、关键事实、当前状态和当前场所的完整摘要。

#### 故事

只展示经过审核、有来源或明确标注编辑属性的内容。机构性格、喜好或昵称叙述必须与事实字段区分。

#### 时间线

展示出生、命名、公开亮相、迁移及其他生命周期事件，并区分发生日期、宣布日期和来源发布日期。

#### 家族

显示父母、子女、全同胞、半同胞及适用的祖辈摘要，同时提供进入家族探索器的入口。

#### 足迹

显示按时间排序的居住记录和迁移事件，同时提供进入地图个体足迹模式的入口。

#### 影像

只展示符合媒体许可规则的内容；无合规媒体时显示统一无图状态。

#### 资料依据与修订

展示来源链、冲突、不确定性、最后核实日期、档案版本、修订记录和适用许可。

### 8.4 档案内导航

- 桌面端可以使用吸顶章节导航。
- 移动端必须提供可操作且不遮挡内容的章节导航形式。
- 章节使用稳定锚点，可直接分享。
- 语言切换保留当前章节。
- 无 JavaScript 时，锚点导航仍可工作。

### 8.5 跨档案探索入口

页面中应提供基于真实关系的下一步：

- 父母；
- 子女；
- 全同胞和半同胞；
- 同一机构的其他熊猫；
- 同一迁移事件的参与个体；
- 关联机构或场所；
- 有明确方法依据的相关个体。

不得使用未发布本地分数、虚构热度或无解释“猜你喜欢”作为推荐依据。

## 9. 档案摘要与独立探索器的分工

### 9.1 档案内摘要必须可完成基础任务

用户不离开个体档案，也必须能够：

- 知道父母和主要子女；
- 理解主要亲缘关系；
- 查看当前场所；
- 阅读居住与迁移顺序；
- 打开相关机构或场所；
- 查看关键来源。

### 9.2 家族探索器承担深度关系任务

用于：

- 多代关系；
- 全同胞和半同胞筛选；
- 两只熊猫的关系路径；
- 代际深度控制；
- 跨档案浏览。

### 9.3 地图探索器承担深度空间任务

用于：

- 全球机构浏览；
- 个体完整足迹；
- 多个迁移节点和参与个体；
- 野生保护分布快照。

### 9.4 上下文连续性

从档案进入探索器时，URL 必须包含适用的：

- 当前熊猫；
- 当前模式；
- 时间范围；
- 关系或筛选；
- 返回章节。

从探索器返回时，应回到原档案和原章节，不丢失任务语义。

## 10. 家族探索器

家族探索器采用 **“关系优先，而不是画布优先”**。

### 10.1 默认入口

可以：

- 从个体档案带入焦点熊猫；
- 从顶层导航进入后选择熊猫；
- 从可分享 URL 恢复焦点和比较状态。

### 10.2 首屏关系摘要

优先显示结构化关系：

- 父亲；
- 母亲；
- 子女；
- 全同胞；
- 半同胞；
- 祖父母；
- 孙辈。

图形谱系与该摘要并列，不得先显示一个无法理解的大型空画布。

### 10.3 图形与列表同步

选择任一熊猫时：

- 图形焦点更新；
- 结构化列表更新；
- 详情摘要更新；
- URL 更新；
- 浏览器返回可以恢复前一焦点。

图形坐标不得创造结构化数据中不存在的关系含义。

### 10.4 两只熊猫的关系

用户可以选择第二只熊猫，查看：

- 最短亲缘路径；
- 每一步关系类型；
- 路径中的每个真实个体；
- 暂定、冲突或未知边。

结果必须以文字和链接表达，不只使用高亮线条。

### 10.5 复杂度控制

- 默认不无限展开全部数据库。
- 使用代际深度、关系类型和按需展开控制范围。
- 影响任务含义的展开状态写入 URL。
- 大型图形资源按需加载。

### 10.6 家族 URL 状态

至少表达：

- 焦点熊猫；
- 代际深度；
- 关系筛选；
- 比较对象。

示例：

```text
/{locale}/lineage?panda=mei-xiang&depth=2&compare=bao-li
```

具体键名可以在实施规格中微调，但语义不得丢失。

## 11. 地图探索器

地图使用统一技术底座，但信息架构分成三个互斥的主模式：

1. **全球机构**
2. **个体足迹**
3. **野生保护**

不得默认把三种语义叠加成一个“万能地图”。

### 11.1 全球机构

这是地图默认模式。

主要对象是机构和场所，而不是把每只熊猫都表现为普通地图点。

选择机构后显示：

- 机构名称和类型；
- 所在地；
- 关联场所；
- 当前熊猫；
- 历史熊猫；
- 最后核实时间；
- 进入机构详情；
- 进入熊猫档案。

### 11.2 个体足迹

必须从一只熊猫出发，或先选择熊猫。

展示：

- 按时间排序的居住记录；
- 迁移事件；
- 事件参与个体；
- 场所与机构关系；
- 当前场所；
- 来源与核实状态。

“宣布将迁移”和“实际完成迁移”必须使用不同状态，不得通过线路暗示已经发生。

地图和时间列表同步，但列表单独即可完成完整任务。

### 11.3 野生保护

展示：

- 保护区；
- 栖息地；
- 分布快照；
- 数据日期；
- 公开精度；
- 位置模糊或安全处理说明。

不得与圈养机构、个体足迹或精确观察点共用含糊图例。首个 Beta 不扩展实时轨迹或精确观察点能力。

### 11.4 地图与列表

- 结构化列表始终存在，不只在地图失败时出现。
- 地图选择与列表选择共享同一状态。
- 列表可以筛选、选择、进入详情和恢复上下文。
- 读屏和键盘用户仅使用列表也能完成全部核心地图任务。

### 11.5 地图 URL 状态

至少表达：

- 模式；
- 选中实体；
- 个体足迹中的熊猫；
- 时间范围；
- 关键筛选。

视口只在以下情况写入 URL：

- 用户主动分享当前地图视图；
- 视口变化决定返回的数据范围；
- 恢复视口对任务有明确意义。

示例：

```text
/{locale}/map?mode=footprint&panda=bao-li&from=2023-01-01&to=2026-07-15
```

临时悬停和动画进度不写入 URL。

## 12. 机构与场所

### 12.1 领域边界

**机构**是动物园、保护机构、研究中心或其他组织实体。

**场所**是熊猫实际居住、隔离、繁育、转运、展示或被公开定位的物理地点。

一个机构可以管理一个或多个场所。熊猫居住记录关联场所，并由场所关联机构。居住在某场所不自动表示该机构拥有熊猫的法律所有权。

### 12.2 机构详情

机构详情提供：

- 中英文名称；
- 别名；
- 机构类型；
- 所在地；
- 官方来源；
- 最后核实时间；
- 当前熊猫；
- 历史熊猫及居住时间范围；
- 关联场所；
- 相关迁移事件；
- 在地图中查看；
- 进入熊猫档案。

### 12.3 场所详情

场所拥有可分享 URL，但首个 Beta 不要求与机构同等完整的编辑页面。

场所页或场所摘要至少提供：

- 名称；
- 所属或管理机构；
- 公开位置精度；
- 与熊猫有关的居住记录；
- 最后核实时间；
- 在地图中查看；
- 进入相关机构和熊猫档案。

### 12.4 搜索和显示规则

- 搜索结果明确标注“机构”或“场所”。
- 熊猫档案中的“当前场所”链接到具体场所。
- 当前场所附近同时显示其所属机构。
- 地图点和图例不得把机构中心、实际场所和野生观察对象混为同一实体类型。

## 13. 我的熊猫

“我的熊猫”是匿名、本地、轻量的回访入口，不是账户系统。

### 13.1 收藏的熊猫

提供：

- 已收藏的真实熊猫档案；
- 进入档案；
- 取消收藏；
- 按名称或最近收藏排序。

不得提供：

- 尚不存在的通知承诺；
- 公开收藏数量；
- 关注人数；
- 社交动态；
- 无来源个性化推荐。

### 13.2 最近浏览

可以包括熊猫、机构和场所。最近浏览以恢复上下文为目标，可返回原实体、章节或来源页面。

必须：

- 设置合理数量上限；
- 支持单项移除；
- 支持全部清除；
- 明确保存范围。

### 13.3 本地数据说明

页面必须说明：

- 数据只保存在当前浏览器；
- 更换设备可能丢失；
- 无痕模式可能不保留；
- 清理浏览器数据会清除收藏和最近浏览。

收藏和最近浏览不得发送到分析系统，不得用于公开排名或广告画像。

### 13.4 无 JavaScript 状态

无 JavaScript 时：

- 页面解释本地收藏不可用；
- 不显示无法操作的收藏按钮；
- 不影响其他档案探索任务；
- 可以继续使用正常导航和搜索。

## 14. 核心公共旅程

### 14.1 首次访问：搜索并确认一只熊猫

1. 用户进入语言化首页。
2. 首屏理解 PandaAtlas 的定位。
3. 使用名称、别名或旧称搜索。
4. 从建议直接进入档案，或进入图鉴完整结果。
5. 在档案首屏确认身份、当前场所、父母和核实时间。
6. 需要时打开来源。
7. 继续进入家族、足迹、机构或相关熊猫。

成功标准：用户不需要先理解地图、数据库结构或产品品牌故事，就能确认目标个体。

### 14.2 从搜索结果继续发现

1. 用户在图鉴中筛选或排序。
2. URL 更新并可分享。
3. 用户进入一个档案。
4. 返回时恢复原查询、筛选、页码和滚动语义。
5. 无精确结果时选择可解释的相关实体。

### 14.3 家族探索

1. 用户从个体档案的家族摘要进入探索器。
2. 焦点熊猫和适用深度写入 URL。
3. 首先读取结构化关系摘要。
4. 按需使用图形谱系。
5. 选择第二只熊猫比较关系路径。
6. 进入路径中的另一个真实档案。
7. 返回时恢复原焦点和比较状态。

### 14.4 个体足迹探索

1. 用户从档案足迹章节进入地图。
2. 地图自动进入个体足迹模式并带入熊猫。
3. 用户通过时间列表理解居住和迁移顺序。
4. 选择场所或事件，地图与列表同步。
5. 打开机构、场所或共同迁移个体。
6. 返回档案时回到足迹章节。

### 14.5 全球机构探索

1. 用户从顶层地图或首页进入全球机构模式。
2. 通过地图或列表筛选机构。
3. 选择机构查看当前和历史熊猫。
4. 进入机构详情或熊猫档案。
5. 返回时恢复原机构、筛选和视图。

### 14.6 来源核验

1. 用户从关键事实旁的轻量来源入口进入。
2. 页面定位到具体来源或资料依据章节。
3. 用户查看来源机构、发布日期、核实日期和结论状态。
4. 返回时回到触发来源的事实位置。

### 14.7 语言切换

1. 用户在任一实体、章节或探索状态切换语言。
2. 进入同一语义对象的目标语言 URL。
3. 保留查询、筛选、模式、焦点、比较和章节。
4. 目标语言内容不完整时显示明确翻译状态。
5. 不得静默跳回首页或清空探索状态。

### 14.8 回访收藏

1. 用户在档案中收藏熊猫。
2. 以后进入“我的熊猫”。
3. 从收藏或最近浏览恢复档案。
4. 页面明确说明本地保存边界。

## 15. URL、历史和状态恢复

### 15.1 必须进入 URL 的状态

#### 图鉴

- 查询词；
- 实体类型；
- 筛选；
- 排序；
- 页码。

#### 个体档案

- 当前章节锚点；
- 具体来源、事实或修订定位。

#### 家族

- 焦点熊猫；
- 代际深度；
- 关系筛选；
- 比较对象。

#### 地图

- 模式；
- 选中实体；
- 时间范围；
- 关键筛选；
- 有任务意义的视口。

#### 机构与场所

- 适用的子区域；
- 时间范围；
- 选中熊猫或记录。

### 15.2 不进入 URL 的状态

- 悬停；
- 动画进度；
- 未提交输入；
- 短暂 tooltip；
- 纯视觉展开且不改变任务语义的状态。

### 15.3 浏览器历史

- 提交搜索、改变主筛选、切换地图模式、切换焦点或进入实体时，应产生可理解的历史状态。
- 高频连续输入和地图拖动不应污染历史记录。
- 前进和后退必须恢复同一任务语义。
- 页面不得只恢复 URL 而丢失与其对应的列表选择或焦点。

### 15.4 无效或过期状态

当 URL 中的对象不存在、被撤回或当前语言不可用时：

- 解释原因；
- 保留仍有效的查询或筛选；
- 提供最近有效实体或父级入口；
- 不无提示清空全部状态；
- 受撤回影响的事实不得从客户端缓存继续展示为当前结论。

## 16. 双语行为

### 16.1 路由等价

中文和英文必须拥有独立、可分享、可索引的规范 URL。核心页面和核心任务在两种语言中功能等价。

### 16.2 切换规则

语言切换保留：

- 实体；
- 查询；
- 筛选；
- 排序；
- 页码；
- 地图模式；
- 选中对象；
- 家族焦点；
- 比较对象；
- 章节和来源定位。

### 16.3 内容不完整

目标语言缺少部分内容时：

- 页面仍停留在同一实体；
- 明确标注翻译状态；
- 已发布结构化事实仍按目标语言界面展示；
- 不自动混入未审核翻译；
- 不跳回首页。

### 16.4 URL 稳定性

- 查询参数和值使用语言无关的机器标识。
- 显示名称可以本地化，slug 不随界面语言临时翻译。
- 若中英文使用不同公开 slug，必须由同一稳定实体映射并互相声明语言替代关系。

## 17. 移动端、键盘、读屏与非视觉等价旅程

### 17.1 移动端

- 保留全部顶层入口。
- 搜索是一级入口。
- 关键事实优先于媒体和故事。
- 地图和谱系先显示结构化摘要，再按需展示图形。
- 筛选、详情和选择不得要求横屏、悬停或精确拖拽。
- 不使用只能在桌面侧栏中完成的关键操作。

### 17.2 键盘

用户必须能通过标准键盘完成：

- 打开和关闭导航；
- 搜索与提交；
- 筛选和排序；
- 选择图鉴结果；
- 跳转档案章节；
- 选择地图列表项；
- 切换地图模式；
- 切换家族焦点；
- 选择比较对象；
- 打开来源；
- 收藏和取消收藏。

焦点顺序跟随任务顺序，不跟随图形坐标。

### 17.3 读屏

页面需要：

- 跳过导航；
- 清晰主标题；
- 搜索、结果、章节、地图列表和家族列表区域名称；
- 动态结果数量的适度通知；
- 关系类型和状态的完整文字；
- 对图形变化不过度打断的状态消息。

### 17.4 非视觉地图

只使用结构化列表，用户也能：

- 选择模式；
- 筛选机构；
- 查看当前和历史熊猫；
- 阅读居住和迁移顺序；
- 查看数据日期和完整性；
- 进入机构、场所和熊猫档案。

### 17.5 非视觉家族

只使用关系列表，用户也能：

- 查看父母、子女、同胞和祖辈；
- 切换焦点；
- 选择比较对象；
- 阅读亲缘路径；
- 理解暂定、冲突和未知关系；
- 进入真实档案。

### 17.6 图形与文字状态统一

图形和结构化界面必须访问同一语义状态。不得出现：

- 图形已选择对象但列表仍停留在旧对象；
- 列表筛选不影响图形；
- 图形展示比列表更多未标注事实；
- 两套界面生成不同关系或位置结论。

## 18. JavaScript 降级与部分失败

### 18.1 无 JavaScript 必须可读

以下内容必须通过服务器输出或普通链接可用：

- 首页定位和主要入口；
- 搜索结果；
- 个体档案关键事实；
- 时间线；
- 家族摘要；
- 足迹列表；
- 来源与修订；
- 机构和场所信息。

### 18.2 高级交互降级

- 高级筛选可以退化为服务器提交表单。
- 地图退化为完整结构化列表。
- 谱系退化为结构化关系列表。
- 章节锚点仍可工作。
- 语言切换使用普通链接。

### 18.3 客户端专属能力

收藏、最近浏览及某些即时图形交互在无 JavaScript 时可以不可用，但必须：

- 明确说明；
- 不显示失效控件；
- 不阻断其他任务；
- 不把失败误报为“没有数据”。

### 18.4 部分失败

当地图、谱系、媒体或客户端数据增强失败时：

- 保留服务器已呈现内容；
- 显示具体受影响能力；
- 提供可继续的结构化路径；
- 不用全页错误覆盖仍可用的档案事实。

## 19. 页面职责边界

| 页面 | 首要任务 | 必须提供 | 不负责 |
| --- | --- | --- | --- |
| 首页 | 理解并进入 | 定位、搜索、真实档案、可信修订 | 完整地图、完整谱系、宏观 Dashboard |
| 图鉴 | 搜索与发现 | 混合实体结果、筛选、排序、URL 状态 | 生成故事、伪热度排名 |
| 个体档案 | 理解一只熊猫 | 关键事实、关系、足迹、来源、修订 | 跨数据库大型图形分析 |
| 家族 | 理解关系 | 结构化关系、谱系图、路径比较 | 替代个体档案 |
| 地图 | 理解空间与迁移 | 三模式、结构化列表、数据状态 | 把不同地理语义叠成一层 |
| 机构 | 理解组织关联 | 当前/历史熊猫、场所、迁移、来源 | 代表具体居住地点本身 |
| 场所 | 理解实际地点 | 居住记录、机构关系、公开精度 | 代表整个机构组织 |
| 我的熊猫 | 回访 | 本地收藏、最近浏览、清除控制 | 账户、社交、通知、画像 |

## 20. IA 验收场景

后续原型、实施和浏览器测试至少覆盖：

1. 中文首页搜索中文名进入规范档案。
2. 英文首页搜索英文名进入英文规范档案。
3. 使用旧名命中当前档案并解释匹配。
4. 无语言旧详情永久重定向且保留章节锚点。
5. 图鉴筛选、刷新、分享和返回恢复同一结果。
6. 个体档案首屏无需先阅读故事即可确认关键身份事实。
7. 从档案家族章节进入探索器并返回原章节。
8. 仅用关系列表完成焦点切换和关系比较。
9. 从档案足迹进入地图个体模式并返回原章节。
10. 仅用地图列表完成机构和迁移探索。
11. 全球机构模式不混入野生保护图例。
12. 宣布迁移不提前改变当前场所。
13. 机构与场所搜索结果明确区分。
14. 语言切换保留实体、章节和探索状态。
15. 目标语言内容不完整时不跳回首页。
16. 无 JavaScript 可阅读档案、家族摘要、足迹和来源。
17. 移动端菜单保留全部顶层入口和搜索。
18. 键盘完成图鉴、档案、家族、地图和来源核心路径。
19. 读屏完成地图与谱系的结构化等价任务。
20. “我的熊猫”明确本地保存边界并支持清除。

## 21. 对后续 Wayfinder 票的约束

### Design the frontend system boundary and token/component architecture

必须为以下稳定信息架构提供清晰所有权：

- 语言化 route shell；
- 全局导航与搜索；
- 品牌页与产品页 register；
- 档案章节模式；
- 图形与结构化列表共享状态；
- URL 状态解析和序列化；
- 机构与场所实体呈现；
- 无 JavaScript 的服务器内容边界。

### Define content, data-honesty, media, and UI-state rules

必须定义本信息架构中每类页面和实体的：

- 已核实、待核实、冲突、过期和 fallback 标记；
- 无结果、无媒体、无地图、无关系和翻译不完整状态；
- 搜索匹配理由；
- 来源、最后核实时间和修订摘要；
- 机构、场所和当前场所的文案边界。

### Prototype the cross-surface PandaAtlas design language

原型必须展示一条完整闭环，而不是孤立页面：

- 首页搜索；
- 图鉴结果；
- 个体档案；
- 家族结构化摘要和图形；
- 地图列表和图形；
- 机构或场所；
- 返回与语言切换；
- 移动端、键盘、fallback 和无媒体状态。

### Define frontend quality gates and visual verification

质量门禁必须验证：

- 规范路由和永久重定向；
- canonical 与 `hreflang`；
- URL 状态恢复；
- 浏览器历史；
- 双语等价；
- 移动导航；
- 无 JavaScript；
- 地图和谱系非视觉等价；
- 机构与场所语义不混淆。

## 22. 非目标

本文不决定：

- 具体组件 API；
- Token 名称和值；
- 数据请求和缓存实现；
- API 与后端领域模型重设计；
- 最终地图供应商；
- 高保真视觉构图；
- 精确筛选字段全集；
- 账户同步、通知、评论或社交功能；
- 专业后台信息架构。

这些内容由后续票或独立产品工作决定，但不得破坏本文的规范路由、核心闭环、领域边界和等价旅程。

## 23. 最终决议

PandaAtlas 的公共信息架构不以“有多少页面和功能入口”为成功标准，而以用户是否能稳定完成以下任务为标准：

> 在任意主要访问条件下，找到一只熊猫，快速确认其身份和关键事实，理解这些信息的来源与状态，沿真实家族、足迹、机构和场所继续探索，并通过可分享、可恢复、可切换语言的 URL 保持上下文。

首页、图鉴、档案、家族、地图、机构和“我的熊猫”因此具有不同职责，但共同组成一个可信、双语、可恢复的档案探索网络。
